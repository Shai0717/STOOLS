# 🛠️ STOOLS v0.2 통합 사용 설명서
**Skript-Base SHAI Multi Tools STOOLS v0.2**
Copyright 2026. Star^N Studio All rights reserved.

특정 블록에 좌클릭이나 우클릭을 했을 때 지정된 동작(명령어 실행 또는 텔레포트)이 작동하도록 설정하는 관리자용 스크립트입니다. 
v0.2부터는 강력한 **통합 GUI**를 지원하여 복잡한 명령어 없이도 마우스 클릭만으로 모든 설정을 제어할 수 있습니다. 
(데이터는 `plugins/stools/data.yml`에 안전하게 저장됩니다.)

---

## ✨ v0.2 주요 기능 및 특징
* **🖥️ 통합 GUI 시스템:** 명령어 암기 없이 `/stools` 하나로 모든 설정을 마우스로 제어합니다.
* **🖱️ 클릭 완벽 구분:** 같은 블록이라도 좌클릭과 우클릭에 각각 다른 동작을 할당할 수 있습니다.
* **👀 조건부 실행 (Sneak):** 플레이어가 웅크린 상태인지 아닌지에 따라 작동 여부를 결정합니다.
* **🚀 동작 지원:** 정밀한 좌표 제어가 가능한 텔레포트(`tp`) 기능과, 관리자 권한 명령어(`cmd`) 대리 실행을 지원합니다.
* **💾 YAML 데이터 연동:** 서버 재시작 시에도 설정된 블록 데이터가 영구적으로 유지됩니다.

## 🖥️ 1. GUI 시스템 안내 (v0.2 신규)

가장 직관적이고 편리하게 STOOLS를 사용하는 방법입니다. 모든 명령어는 **OP 권한**이 필요합니다.

* **`/stools`**: STOOLS 메인 GUI를 엽니다.
* **`/stools help`** 또는 **`/?`**: 사용법 도움말을 확인합니다.

### 📍 CLICK GUI 메뉴 (블록 설정)
메인 GUI에서 **커맨드 블록(STOOLS CLICK GUI)**을 클릭하면 다음 메뉴가 나타납니다.
1. **Set TP (레드스톤 램프):** 블록 클릭 시 이동할 텔레포트 좌표를 설정합니다.
   * *GUI 좌표 컨트롤러:* X, Y, Z 좌표를 클릭하여 `+1`부터 `+1,000`까지 미세 조정할 수 있으며, **Shift 클릭 시 최대 `10,000,000` 단위**로 빠르게 설정 가능합니다. 월드(World) 및 웅크리기(Sneak) 조건도 클릭으로 변경할 수 있습니다.
2. **Set Command (커맨드 블록):** 블록 클릭 시 실행할 명령어를 설정합니다.
   * *채팅 입력 연동:* 설정을 완료하면 채팅창에 `/stools:input [명령어]`를 입력하여 실행할 명령어를 확정합니다.
3. **Delete (TNT):** 설정된 블록의 기능을 제거 모드로 전환합니다.
4. **List (종이):** SDIR 시스템과 연동하여 현재 설정된 블록 데이터 목록을 확인합니다.

> **💡 GUI 설정 마무리:** GUI에서 설정을 마친 후 메뉴를 닫고, **기능을 부여할 블록을 직접 클릭(좌/우클릭)**해야 최종적으로 바인딩이 완료됩니다.

## 📜 2. 명령어(CLI) 안내 (숙련자용)

GUI를 거치지 않고 기존 v0.1 방식처럼 명령어로 빠르게 설정할 수도 있습니다. 

> **기본 구조:** `/stools:click set [웅크리기_조건] [동작_유형] [추가_설정값...]`

**[웅크리기 조건]**
* `none` (또는 `n`): 웅크리기 여부와 상관없이 항상 작동
* `sneak` (또는 `s`): 웅크린(Shift) 상태에서만 작동
* `notsneak` (또는 `ns`): 웅크리지 않은 상태에서만 작동

#### 📍 텔레포트 (Teleport) 설정
* **명령어:** `/stools:click set [조건] tp [월드명] [X Y Z]`
* **설명:** 블록 클릭 시 지정한 월드의 좌표로 이동합니다. (좌표는 띄어쓰기로 구분)
* **예시:** `/stools:click set n tp world 100 64 -200`

#### ⚡ 명령어 (Command) 설정
* **명령어:** `/stools:click set [조건] cmd [명령어]`
* **설명:** 블록 클릭 시 지정한 명령어를 **OP 권한**으로 실행합니다. (슬래시 `/` 제외)
* **예시:** `/stools:click set s cmd give %player% diamond 1`

#### 🗑️ 설정 제거 (Delete) 및 확인 (List)
* **삭제:** `/stools:click delete` (입력 후 지울 블록을 좌/우클릭)
* **목록:** `/stools:click list` (SDIR 파일 매니저 연동 필수)

## 💡 3. 사용 예제 (따라 하기)

**Q. 특정 스펀지 블록을 우클릭하면 스폰(spawn) 명령어가 쳐지게 하고 싶어요!**
1. **[GUI 방식]** `/stools` 접속 ➔ CLICK GUI ➔ Set Command ➔ 조건 설정 후 완료 ➔ 채팅창에 `/stools:input spawn` 입력
2. **[CLI 방식]** 채팅창에 `/stools:click set none cmd spawn` 입력
3. 시스템 메시지 `"블럭을 클릭 하여 지정한 값의 설정을 완료하세요."`를 확인합니다.
4. 원하는 스펀지 블록을 **우클릭**합니다.
5. `"성공적으로 설정 되었습니다."` 메시지가 뜨면 완료!

**Q. 잘못 설정했어요. 지우고 싶어요.**
1. 채팅창에 `/stools:click delete` 를 입력하거나, GUI에서 **Delete (TNT)** 메뉴를 누릅니다.
2. 잘못 설정했던 블록을 다시 **우클릭**합니다. (우클릭 설정을 지울 경우)
3. `"성공적으로 삭제 되었습니다."` 메시지가 뜨면 삭제 완료!

## ⚠️ 4. 주의 사항 및 참고
1. **좌/우클릭 독립성:** 같은 블록이더라도 좌클릭과 우클릭에 각기 다른 동작(예: 좌클릭은 상점 열기, 우클릭은 스폰 이동)을 덮어씌움 없이 동시에 설정할 수 있습니다.
2. **좌표 입력 주의:** 명령어(CLI)로 텔레포트 설정 시 X, Y, Z 좌표는 반드시 **띄어쓰기 한 칸**으로 구분해야 합니다. (예: `100 64 100`) 문자가 섞이면 오류가 발생합니다. (※ GUI 사용 시 이 문제가 발생하지 않습니다.)
3. **덮어쓰기 불가:** 이미 설정된 클릭 방식에 새로운 설정을 덮어씌울 수 없습니다. 반드시 **설정 제거(Delete)** 모드로 기존 설정을 지운 후 다시 설정해야 합니다.
