# 🛠️ STOOLS v0.3 통합 사용 설명서
**Skript-Base SHAI Multi Tools STOOLS v0.3**
Copyright 2026. Star^N Studio All rights reserved.

특정 블록에 좌클릭이나 우클릭을 했을 때 지정된 동작(명령어 실행, 텔레포트, 워프)이 작동하도록 설정하고, 서버 내의 워프(Warp) 지점들을 체계적으로 관리할 수 있는 **올인원 관리자용 스크립트**입니다. 
v0.3부터는 완전히 새로워진 **워프(Warp) 시스템**과 최신 Skript 2.15.0+ 완벽 호환 패치가 적용되었습니다.
(데이터는 `plugins/stools/click_data.yml` 및 `warp_data.yml`에 안전하게 분리 저장됩니다.)

---

## ✨ v0.3 주요 기능 및 특징
* **🌌 완벽한 워프(Warp) 시스템 (신규):** 플레이어의 시선 방향(Yaw/Pitch)까지 정확히 저장하며, 워프 이동과 동시에 특정 명령어가 실행되도록 설정할 수 있습니다.
* **🖥️ 직관적인 3중 GUI 시스템:** 명령어 암기 없이 `/stools` 하나로 파일 관리(SDIR), 블록 클릭(CLICK), 워프(WARP) 관리를 모두 마우스로 제어합니다.
* **🖱️ 클릭 완벽 구분 및 조건부 실행:** 같은 블록이라도 좌클릭/우클릭을 구분하며, 웅크리기(Sneak) 상태에 따라 동작 여부를 세밀하게 결정합니다.
* **🔗 강력한 시스템 시너지:** 설정해둔 워프 지점을 '블록 클릭' 액션에 그대로 연결할 수 있습니다.
* **🛡️ 최신 버전 호환 방어:** Paper 26.1.x의 JSON 텍스트 처리 변경으로 인한 GUI 먹통 현상을 완벽하게 해결했습니다.

## 🖥️ 1. GUI 시스템 안내 (가장 쉬운 방법)

복잡한 명령어 대신 마우스 클릭만으로 모든 것을 설정하세요. (모든 기능은 **OP 권한**이 필요합니다.)

* **`/stools`**: STOOLS 통합 메인 GUI를 엽니다.
* **`/stools help`** 또는 **`/?`**: 사용법 도움말을 확인합니다.

### 📍 CLICK GUI 메뉴 (블록 클릭 액션 설정)
메인 GUI에서 **커맨드 블록**을 누르면 진입합니다.
1. **Set TP (레드스톤 램프):** 클릭 시 이동할 X, Y, Z 좌표를 정밀하게 설정합니다. (Shift 클릭 시 단위 증폭)
2. **Set Command (커맨드 블록):** 클릭 시 실행될 명령어를 설정합니다. (입력 대기 모드 지원)
3. **Set Warp (화살표 - 신규!):** 클릭 시 지정된 '워프'로 이동하게 만듭니다.
4. **Delete (TNT):** 설정된 블록의 기능을 지우는 제거 모드로 전환합니다.
5. **List (종이):** SDIR 매니저를 열어 현재 블록 데이터(`click_data.yml`)를 직접 확인합니다.

### 🌌 WARP GUI 메뉴 (워프 관리 - 신규!)
메인 GUI에서 **화살표**를 누르면 진입합니다. 등록된 모든 워프 목록이 종이 형태로 표시됩니다.
1. **Set Warp (화살표):** 현재 서 있는 위치를 기반으로 새로운 워프를 등록합니다.
2. **워프 상세 편집 (워프 아이콘 클릭):**
   * **Delete (TNT):** 해당 워프를 삭제합니다.
   * **Edit Location (가위):** 해당 워프의 위치를 '현재 내 위치'로 덮어씁니다.
   * **Edit Command (가위):** 해당 워프로 이동할 때 실행될 명령어를 추가/수정/삭제합니다.
   * **Run Warp!! (엔더 진주):** 해당 워프로 즉시 이동하여 테스트해 봅니다.

> **💡 GUI 설정 마무리:** 클릭 액션(CLICK GUI)에서 설정을 마친 후 메뉴를 닫고, **기능을 부여할 블록을 직접 클릭(좌/우클릭)**해야 최종적으로 블록에 마법이 부여됩니다!

## 📜 2. 명령어(CLI) 안내 (숙련자용)

GUI를 열지 않고 채팅창에서 명령어로 빠르게 세팅하는 방법입니다.

### 🖱️ 클릭(Click) 시스템 명령어
> **기본 구조:** `/stools:click set [웅크리기_조건] [동작_유형] [추가_설정값...]`
> * **웅크리기 조건:** `none`(상관없음), `sneak`(웅크릴때만), `notsneak`(서있을때만)

* **텔레포트 설정:** `/stools:click set [조건] tp [월드명] [X Y Z]`
* **명령어 대리 실행:** `/stools:click set [조건] cmd [명령어]`
* **워프 이동 (신규!):** `/stools:click set [조건] warp [워프별칭]`
* **삭제 및 목록:** `/stools:click delete` (입력 후 블록 클릭) / `/stools:click list`

### 🌌 워프(Warp) 시스템 명령어 (신규!)
* **워프 생성:** `/stools:warp set [별칭] [선택:명령어]` (현재 위치를 워프로 저장)
* **워프 삭제:** `/stools:warp delete [별칭]`
* **위치 덮어쓰기:** `/stools:warp edit [별칭] [선택:명령어]` (현재 위치로 갱신)
* **명령어만 수정:** `/stools:warp editcmd [별칭] [명령어]` (명령어를 안 적으면 기존 명령 삭제)
* **워프 이동:** `/stools:warp run [별칭]`
* **워프 목록:** `/stools:warp list`

## 💡 3. 사용 예제 (이렇게 써보세요!)

**Q. 지금 서 있는 곳을 '상점'으로 만들고, 워프할 때마다 안내 문구를 띄우고 싶어요!**
1. 상점 한가운데에 섭니다.
2. 채팅창에 `/stools:warp set 상점 say 상점에 오신 것을 환영합니다!` 라고 입력합니다.
3. `"성공적으로 ... 지정하였습니다."` 메시지가 뜨면 워프 완성!

**Q. 특정 스펀지 블록을 우클릭하면, 방금 만든 '상점'으로 이동하게 하고 싶어요!**
1. **[GUI 방식]** `/stools` ➔ CLICK GUI ➔ Set Warp (화살표) ➔ 조건 설정(완료) ➔ 채팅창에 `/stools:input 상점` 입력
2. **[CLI 방식]** 채팅창에 `/stools:click set none warp 상점` 입력
3. 시스템 메시지 `"블럭을 클릭 하여..."`를 확인한 뒤, 원하는 스펀지 블록을 **우클릭**합니다.
4. `"성공적으로 설정 되었습니다."` 메시지가 뜨면 완료! 이제 우클릭하면 상점으로 이동합니다.

## ⚠️ 4. 필수 주의 사항 및 참고
1. **커스텀 플러그인 필수:** STOOLS v0.3은 Skript 2.15.0+ 환경에서 작동하도록 설계되었습니다. 반드시 SDIR 및 GUI가 정상 작동하도록 패치된 전용 **`skUtilities-0.9.3-PATCHED.jar`** 파일이 서버 플러그인 폴더에 있어야 합니다.
2. **워프 별칭 주의 사항:** 워프의 이름(별칭)에는 마침표(`.`)를 사용할 수 없습니다. (데이터 구조 엉킴 방지)
3. **좌표 띄어쓰기:** CLI로 텔레포트를 설정할 때 `X Y Z`는 반드시 띄어쓰기 한 칸으로 구분해야 합니다.
4. **덮어쓰기 불가 (클릭 액션):** 이미 클릭 액션이 부여된 블록에 새로운 액션을 덮어씌울 수 없습니다. 반드시 **설정 제거(Delete)** 모드로 기존 기능을 지운 후 다시 부여해 주세요.
