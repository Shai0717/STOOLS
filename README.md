# 🛠️ STOOLS:click 기술 매뉴얼
Skript-Base SHAI Tools STOOLS v0.1
Copyright 2026. Star^N Studio All rights reserved.

특정 블록에 좌클릭이나 우클릭을 했을 때 지정된 동작(명령어 실행 또는 텔레포트)이 작동하도록 설정하는 관리자용 스크립트입니다. 데이터는 YAML 파일(`plugins/stools/data.yml`)에 안전하게 저장됩니다.

## ✨ 주요 기능
* **클릭 구분:** 좌클릭과 우클릭에 각각 다른 동작을 할당할 수 있습니다.
* **조건부 실행 (Sneak):** 플레이어가 웅크린 상태인지 아닌지에 따라 작동 여부를 결정할 수 있습니다.
* **동작 지원:** 지정된 좌표로 텔레포트(`tp`)하거나, 특정 명령어(`cmd`)를 실행할 수 있습니다.
* **YAML 데이터 연동:** 서버 재시작 시에도 설정된 블록 데이터가 유지됩니다.

---

## 📜 명령어 안내

모든 명령어는 OP 권한(`permission: op`)이 있어야 사용할 수 있으며, 게임 내에서만 실행 가능합니다.

### 1. 블록에 기능 설정 (Set)
명령어를 입력한 후, **원하는 블록을 클릭(좌클릭 또는 우클릭)**하여 설정을 완료합니다. 클릭한 버튼에 따라 해당 클릭에 기능이 할당됩니다.

> **기본 구조:** `/stools:click set [웅크리기_조건] [동작_유형] [추가_설정값...]`

**[웅크리기 조건]**
* `none` (또는 `n`): 웅크리기 여부와 상관없이 항상 작동
* `sneak` (또는 `s`): 웅크린(Shift) 상태에서만 작동
* `notsneak` (또는 `ns`): 웅크리지 않은 상태에서만 작동

#### 📍 텔레포트 (Teleport) 설정
* **명령어:** `/stools:click set [조건] tp [월드명] [X Y Z]`
* **설명:** 블록 클릭 시 지정한 월드의 좌표로 이동합니다. 좌표는 띄어쓰기로 구분하여 입력합니다.
* **예시:** `/stools:click set n tp world 100 64 -200`
  *(웅크리기 상관없이 클릭 시 world 월드의 100, 64, -200 좌표로 텔레포트)*

#### ⚡ 명령어 (Command) 설정
* **명령어:** `/stools:click set [조건] cmd [명령어] [추가인자]`
* **설명:** 블록 클릭 시 지정한 명령어를 **OP 권한**으로 실행합니다. (슬래시 `/` 제외)
* **예시:** `/stools:click set s cmd give %player% diamond 1`
  *(웅크린 상태로 클릭 시 해당 플레이어에게 다이아몬드 1개 지급)*

---

### 2. 블록 설정 제거 (Delete)
블록에 설정된 기능을 삭제합니다. 좌클릭 설정과 우클릭 설정이 별개로 관리되므로, 삭제할 때도 **지우고 싶은 클릭 방식(좌클릭/우클릭)으로 블록을 클릭**해야 합니다.

* **명령어:** `/stools:click delete` (또는 `d`)
* **사용법:** 명령어 입력 후 기능이 설정된 블록을 좌클릭 또는 우클릭합니다.

---

### 3. 설정 목록 확인 (List)
* **명령어:** `/stools:click list` (또는 `l`)
* **설명:** `data.yml` 파일에 저장된 설정 목록을 확인합니다. (※ `sdir` 플러그인/명령어가 서버에 있어야 정상 작동합니다.)

## 💡 사용 예제 (따라 하기)

**Q. 특정 스펀지 블록을 우클릭하면 스폰(spawn) 명령어가 쳐지게 하고 싶어요!**
1. 채팅창에 `/stools:click set none cmd spawn` 을 입력합니다.
2. 시스템 메시지 `"블럭을 클릭 하여 지정한 값의 설정을 완료하세요."`를 확인합니다.
3. 원하는 스펀지 블록을 **우클릭**합니다.
4. `"성공적으로 설정 되었습니다."` 메시지가 뜨면 완료!

**Q. 잘못 설정했어요. 지우고 싶어요.**
1. 채팅창에 `/stools:click delete` 를 입력합니다.
2. 잘못 설정했던 블록을 다시 **우클릭**합니다. (우클릭 설정을 지울 경우)
3. `"성공적으로 삭제 되었습니다."` 메시지가 뜨면 삭제 완료!

## ⚠️ 주의 사항 및 참고
1. **좌/우클릭 독립성:** 같은 블록이더라도 좌클릭과 우클릭에 각기 다른 동작(예: 좌클릭은 상점 열기, 우클릭은 스폰 이동)을 덮어씌움 없이 동시에 설정할 수 있습니다.
2. **좌표 입력 주의:** 텔레포트 설정 시 X, Y, Z 좌표는 반드시 **띄어쓰기 한 칸**으로 구분해야 합니다. (예: `100 64 100`) 문자가 섞이면 오류가 발생합니다.
3. **덮어쓰기 불가:** 이미 설정된 클릭 방식에 새로운 설정을 덮어씌울 수 없습니다. 반드시 `/stools:click delete`로 기존 설정을 지운 후 다시 설정해야 합니다.
